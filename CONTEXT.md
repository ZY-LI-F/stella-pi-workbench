# Stella 工作流领域词汇

Stella 是监督多个 Pi 执行角色的桌面工作台。用户把一项可交付工作写成“任务”，选择一套固定“流程模板”后分发；Stella 依次唤起对应角色，并把真实运行、产物和失败显示在看板上。

- **任务（Task）**：看板上长期存在的一项工作，包含目标、项目、优先级和验收标准。
- **执行角色（Agent Definition）**：固定且带版本的职责配置，例如侦察员、规划师、构建者；它不是正在运行的进程。
- **Agent 运行（Agent Run）**：某个执行角色为一个流程步骤启动的一次独立 Pi 会话。
- **团队模板（Team Definition）**：一组固定角色席位，表达谁负责什么，不代表所有角色同时运行。
- **流程模板（Workflow Definition）**：按顺序排列的 Agent 步骤和人工关卡。
- **流程实例（Workflow Run）**：某个任务一次真实分发后生成的流程快照与运行记录。
- **步骤实例（Step Run）**：流程实例中一个步骤的状态、时间、会话和结果。
- **产物（Artifact）**：Agent 步骤完成后保留的最终输出，可供后续角色和人工验收查看。
- **人工关卡（Human Gate）**：只有用户明确批准或驳回后才能继续的步骤。
- **分发（Dispatch）**：为任务创建流程实例并开始执行。

“看板列”只表达任务所处阶段；“会话”特指 Pi 会话；“团队”不作为“流程”的同义词。
